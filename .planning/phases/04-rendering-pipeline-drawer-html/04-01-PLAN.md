**CRITICAL ID-threading rule (Pitfall 1, 2 — read 04-RESEARCH.md lines 290-296 before implementing):** Each panel's `$panel_id` serves three roles:
1. It is the `data-panel-id` on that panel's own `<div>`.
2. It is the `$ancestor_panel_id` passed to every `render_item()` inside it — so when an item has children, the child panel's back button `data-back-target` references THIS panel.
3. It is NOT the `data-target` on items' chevrons — that is the child's freshly-generated `$child_panel_id`.

**Method: `private static function render_panel( array $items, array $settings, string $panel_id, bool $is_root ): void`**

Emits one panel div + its items. Structure:
```
// Panel wrapper attributes vary by $is_root.
$classes = 'ddmm-panel';
if ( $is_root ) {
    $classes .= ' ddmm-panel--active';  // D-26: root is active.
}
// Root: no aria-hidden (active). Child: aria-hidden="true" (D-24).
$aria_hidden = $is_root ? '' : ' aria-hidden="true"';
// Child: aria-labelledby set by caller via $panel-labelledby (see render_item).
```
IMPORTANT: For child panels, `aria-labelledby` must point to the back-row title span ID. To thread this, change the signature to `render_panel( array $items, array $settings, string $panel_id, bool $is_root, string $title_id = '' )`. For root, `$title_id` is empty → no aria-labelledby. For child panels, emit `aria-labelledby="{esc_attr($title_id)}"` (D-22, Pitfall 4).

Panel wrapper:
```

    esc_attr( $classes ),
    esc_attr( $panel_id ),
    $title_id ? ' aria-labelledby="' . esc_attr( $title_id ) . '"' : '',  // D-22
    $aria_hidden
```

  When `$is_root` is false, generate `$title_id = uniqid( 'ddmm-back-title-', false );` INSIDE render_item (the caller) and pass it here so both the panel's aria-labelledby and the back-row span id match.

**Method: `public static function render_editor_preview( array $tree, array $settings ): void`** (D-18, DRAW-01 caller in Plan 02)

This is the Elementor edit-mode entry point. It emits ONLY a static inline root panel `<ul>` so the editor user sees the configured items with icons + chevrons. It emits NO overlay, NO drawer wrapper, NO off-canvas transform, NO child panel siblings, and NO back rows (sub-panels are omitted per D-18). Structure:

```php
public static function render_editor_preview( array $tree, array $settings ): void {
    if ( empty( $tree ) ) {
        return;
    }
    echo '<ul class="ddmm-menu">';
    foreach ( $tree as $node ) {
        self::render_editor_item( $node, $settings );
    }
    echo '</ul>';
```

```php
private static function render_editor_item( array $node, array $settings ): void {
    echo '<li class="ddmm-menu__item">';
    // Icon (same render_icon helper — D-30).
    $icon_html = self::render_icon( $node['icon'] ?? [] );
    echo $icon_html; // phpcs:ignore WordPress.Security.EscapeOutput.OutputNotEscaped -- Icons_Manager output is pre-escaped

    // Label <a> (D-01 / D-04) — identical to render_item step 3.
    $url         = $node['url'] ?? '#';
    $target_attr = ! empty( $node['target'] ) ? ' target="_blank"' : '';
    $classes_attr = '';
    if ( ! empty( $node['classes'] ) && is_array( $node['classes'] ) ) {
        $sanitized = array_filter( array_map( 'sanitize_html_class', $node['classes'] ) );
        if ( ! empty( $sanitized ) ) {
            $classes_attr = ' class="' . esc_attr( implode( ' ', $sanitized ) ) . '"';
        }
    }
    printf(
        '<a href="%s"%s%s>%s</a>',
        esc_url( $url ),
        $target_attr,
        $classes_attr,
        esc_html( $node['title'] )
    );

    // Chevron visual ONLY when has_children (D-18 editor preview — no data-target wiring, no panel sibling).
    if ( ! empty( $node['has_children'] ) ) {
        printf(
            '<button type="button" class="ddmm-chevron" aria-label="%s"></button>',
            esc_attr( sprintf( __( 'Show %s submenu', 'devsroom-drilldown-mobile-menu' ), $node['title'] ) )
        );
    }

    echo '</li>';
}
```
