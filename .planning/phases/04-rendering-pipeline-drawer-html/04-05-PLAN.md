<objective>
Close the Phase 4 ARIA gap (VERIFICATION.md WR-01): child panels currently emit
`aria-labelledby="$title_id"` unconditionally in `render_panel()`, but the
referenced `<span id="$title_id">` only renders in `render_back_row()` when
`show_back_title === 'yes'`. When a user disables "Show Parent Name in Back Row",
every child panel carries a dangling `aria-labelledby` reference to a
non-existent DOM node — breaking the ARIA label association and tripping
axe/Lighthouse audits.

Purpose: Restore the ARIA label-association contract in ALL configurations
without losing the user's ability to hide the visible parent name.

Output: Aligned `render_back_row()` that ALWAYS emits the title span (so
`aria-labelledby` always resolves) but switches the span's CSS class between
visible `ddmm-back__title` and visually-hidden `ddmm-back__title screen-reader-text`
based on the `show_back_title` setting. A self-contained `.screen-reader-text`
rule is added to the plugin CSS so the class works regardless of whether the
theme provides one.
</objective>

private static function render_back_row( string $parent_title, string $ancestor_panel_id, string $title_id, array $settings ): void {
    echo '<div class="ddmm-back">';

    // D-10: back button carries data-back-target = ancestor panel id.
    printf(
        '<button type="button" class="ddmm-back__button" data-back-target="%s">&larr; %s</button>',
        esc_attr( $ancestor_panel_id ),
        esc_html__( 'Back', 'devsroom-drilldown-mobile-menu' )
    );

    // D-12: show parent name title, default ON.
    $show_back_title = $settings['show_back_title'] ?? 'yes';
    if ( 'yes' === $show_back_title ) {
        printf(
            '<span class="ddmm-back__title" id="%s">%s</span>',
            esc_attr( $title_id ),
            esc_html( $parent_title )
        );
    }

    echo '</div>';
}

## Chosen approach: always-emit span + screen-reader-text (recommended)

The VERIFICATION.md offers two fixes. We choose the **screen-reader-text approach**
because it preserves the ARIA label association in ALL configurations — when the
toggle is OFF, screen readers still announce the parent name when the panel opens.
This is strictly better for accessibility than gating `aria-labelledby` off, which
would strip the announcement entirely.
