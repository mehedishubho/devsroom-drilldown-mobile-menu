## Menu Source Control Strategy

| Option | Description | Selected |
|--------|-------------|----------|
| Add toggle now | Add "Menu Source" SELECT with WP Menu / Custom Builder options. Phase 3 adds repeater under second option. | ✓ |
| WP dropdown only | Just add WP Menu dropdown, no source toggle. Phase 3 restructures controls later. | |

**User's choice:** Add toggle now (Recommended)
**Notes:** Adding the toggle now avoids restructuring Content Tab controls when Phase 3 adds Custom Builder. Cleaner UX — users see one source selector from the start.

## Content Tab Section Layout

| Option | Description | Selected |

## Tree Node Data Contract

### Panel ID generation

| Option | Description | Selected |
|--------|-------------|----------|
| Pure data, no panel IDs | Tree builder returns data only. Panel IDs generated by Phase 4 renderer. | ✓ |
| Include panel IDs in tree | Tree builder generates panel_id via uniqid. Tighter coupling but simpler renderer. | |

**User's choice:** Pure data, no panel IDs (Recommended)
**Notes:** Clean separation between data layer and rendering layer. Tree builder is pure PHP logic with no HTML/attribute concerns.

### Node field set

| Option | Description | Selected |
|--------|-------------|----------|
| Minimal set | id, title, url, target, classes, has_children, children | ✓ |
| Full WP fields | All WP nav menu item fields including description, attr_title, xfn, object, etc. | |

**User's choice:** Minimal set (Recommended)
**Notes:** Extra fields available from WP source data if needed later. Keeps tree lightweight and focused.

## Empty/No Menu Handling

| Option | Description | Selected |
|--------|-------------|----------|
| Render nothing + editor hint | Zero HTML output on frontend. Editor shows "Select a menu to display" hint. | ✓ |
| Show message everywhere | Always render wrapper with "No menu items" message. Visible on frontend too. | |

**User's choice:** Render nothing + editor hint (Recommended)
**Notes:** Clean frontend behavior — no broken markup when unconfigured. Editor hint ensures user knows widget is present.
