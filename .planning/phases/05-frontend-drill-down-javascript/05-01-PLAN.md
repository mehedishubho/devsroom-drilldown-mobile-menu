<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Elementor settings → rendered HTML | User-configured values (anim type, duration, easing, search placeholder) cross from the editor into the frontend DOM as class names, attribute values, and inline style values |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-01 | Tampering / Spoofing | `animation_type` SELECT value emitted into `class` and `data-ddmm-anim` attr | mitigate | The SELECT control is constrained to 4 enumerated values (slide/fade/scale/slidefade) at the Elementor level — user cannot inject arbitrary strings through the editor UI. Pass through `esc_attr()` in the printf. A crafted value via DB tampering would render as an inert unknown class (no CSS match = no animation type applied, drawer still functions via fallback). Low risk. |
| T-05-02 | Tampering | `animation_easing` value emitted into inline `style` attribute | mitigate | SELECT constrained to 5 enumerated values (ease/ease-in/ease-out/ease-in-out/linear). Pass through `esc_attr()`. An invalid value would produce an invalid CSS declaration that browsers silently drop — no XSS surface because the value is quoted inside the style attribute and cannot break out (no quotes/semicolons in the allowed values). |
| T-05-03 | Tampering | `animation_duration` value cast to int | mitigate | `(int) $duration_size` cast in the printf guarantees a numeric output regardless of input — no string injection possible. The SLIDER control itself constrains to 100-2000. |
| T-05-04 | Spoofing | Close-behavior toggles (auto-open, close-link, close-overlay) as `true`/`false` strings | accept | Ternary `$x ? 'true' : 'false'` produces only these two literal strings — no user data flows through. Zero injection surface. |

**ASVS V5 (Input Validation) compliance:** All four animation/search/drawer-settings values are constrained at the Elementor control layer (SELECT enums, SLIDER numeric range, SWITCHER binary) and re-validated at render via `esc_attr()` (strings) or `(int)` cast (duration). No user-controlled free-text value reaches an HTML context without escaping. The `search_placeholder` TEXT control is the only free-text value and it is rendered by Plan 02's DrawerRenderer (not this plan) — Plan 02 escapes it via `esc_attr()`.
</threat_model>
