Purpose: WordPress calls `uninstall.php` (not the plugin's own code) when an admin clicks Delete. Without it, WP falls back to no-op or `register_uninstall_hook()`. A dedicated `uninstall.php` is the canonical, forward-compatible cleanup location. This makes the uninstaller a production-ready cleanup STUB: it guards against direct access, defines a structured (currently empty) options list, handles multisite, and fires an extensibility hook so future versions only need to add option keys.

Output: `uninstall.php` at plugin root (sibling of `devsroom-drilldown-mobile-menu.php`).

  <action>
Create `uninstall.php` at the plugin root (same directory as `devsroom-drilldown-mobile-menu.php`). This file runs in WordPress's uninstall context — OUTSIDE the plugin's namespace and autoloader. It MUST be plain procedural PHP with no `use` statements, no `\Devsroom_DDMM\` references, and no class references.

Structure the file in this exact order:

1. **Opening `<?php` tag** (no closing `?>` — WordPress coding standard for files that are pure PHP, avoids trailing-whitespace "headers already sent" errors).

2. **File header DocBlock** matching the style of `devsroom-drilldown-mobile-menu.php`. Use a block comment with these fields:
   ```
   /**
    * Uninstaller for Devsroom DrillDown Mobile Menu.
    *
    * Runs when the plugin is deleted via the WordPress admin "Plugins" screen.
    * WordPress core sets the WP_UNINSTALL_PLUGIN constant immediately before
    * including this file — it is NOT defined during normal requests or direct access.
    *
    * @package Devsroom_DDMM
    *
    * Plugin Name: Devsroom DrillDown Mobile Menu
    * Version:     0.0.01
    * Author:      MEHEDI HASSAN SHUBHO
    * Text Domain: devsroom-drilldown-mobile-menu
    */
   ```
   Use `@package Devsroom_DDMM` to match `src/Plugin.php`'s DocBlock package tag (per CLAUDE.md PSR-4 namespace `Devsroom_DDMM`).

3. **Direct-access guard — FIRST executable line after the DocBlock** (before ANY WordPress function call):
   ```php
   if ( ! defined( 'WP_UNINSTALL_PLUGIN' ) ) {
       exit;
   }
   ```
   This is the canonical WP uninstall guard. `WP_UNINSTALL_PLUGIN` is set by WP core ONLY on a legitimate delete. Using `ABSPATH` here would be WRONG — `ABSPATH` is defined on every front-end and admin request, so it would not prevent direct browser access to the file; `WP_UNINSTALL_PLUGIN` is the correct, stronger guard.

4. **Structured options list** — define as a local array, currently empty but structured for forward-compatibility. Add an inline comment explaining that the array is intentionally empty and how to extend it:
   ```php
   /**
    * Plugin options to delete on uninstall.
    *
    * The plugin currently stores no options. To add cleanup for a future option,
    * append its key to this array — the deletion loop below handles the rest
    * automatically (single site AND multisite).
    *
    * @var string[]
    */
   $devsroom_drilldown_options = array(
       // 'devsroom_drilldown_mobile_menu_settings',
       // 'devsroom_drilldown_mobile_menu_version',
   );
   ```

5. **Single-site deletion** — loop over the array and call `delete_option()` for each:
   ```php
   foreach ( $devsroom_drilldown_options as $option_key ) {
       delete_option( $option_key );
   }
   ```

6. **Multisite deletion** — wrapped in `if ( is_multisite() )`. Use the global `$wpdb` (declare `global $wpdb;` — only inside this branch, since single-site never needs it). Iterate all public + archived blog IDs via `get_sites()`, switch to each blog with `switch_to_blog( $blog_id )`, delete the options, then `restore_current_blog()`. Also delete from the main site (site options) via `delete_site_option()`:
   ```php
   if ( is_multisite() ) {
       global $wpdb;

       // Delete options from every blog on the network.
       $site_ids = get_sites(
           array(
               'fields'                 => 'ids',
               'number'                 => 0, // All sites, no limit.
               'update_site_cache'      => false,
               'update_site_meta_cache' => false,
           )
       );

       foreach ( $site_ids as $site_id ) {
           switch_to_blog( (int) $site_id );

           foreach ( $devsroom_drilldown_options as $option_key ) {
               delete_option( $option_key );
           }
       }

       restore_current_blog();

       // Also clean up any network-wide (site) options.
       foreach ( $devsroom_drilldown_options as $option_key ) {
           delete_site_option( $option_key );
       }
   }
   ```
   Notes:
   - `get_sites( array( 'fields' => 'ids', 'number' => 0 ) )` is the modern WP 4.6+ API (WP 6.5+ baseline guarantees it) and returns an array of integer site IDs without loading full WP_Site objects — efficient and allocation-light.
   - `switch_to_blog()` / `restore_current_blog()` MUST be balanced — the loop switches N times, `restore_current_blog()` restores the original exactly once. This is the documented correct pattern.
   - `delete_site_option()` handles the network-wide options table (`wp_sitemeta`) which `delete_option()` per-blog does not touch.
   - Declare `global $wpdb;` even though this branch uses `get_sites()` (not `$wpdb` directly) — keep the `global $wpdb;` declaration because (a) the constraints require it, (b) it documents that this branch operates in a global-state context distinct from single-site, and (c) future maintainers adding direct `$wpdb` queries will not have to discover the missing global.

7. **Extensibility hook — LAST executable line** — fire `do_action()` so third-party add-ons or future plugin code can run custom cleanup:
   ```php
   /**
    * Fires after Devsroom DrillDown Mobile Menu's own data has been removed.
    *
    * Third-party integrations or future plugin modules can hook here to clean
    * up their own data (custom post types, taxonomies, transients, etc.).
    *
    * @since 0.0.01
    */
   do_action( 'devsroom_drilldown_mobile_menu_uninstall' );
   ```

- No closing `?>` tag.
- Match the 4-space indentation and array-item-aligned `=>` style used in `devsroom-drilldown-mobile-menu.php` and `src/Plugin.php`.
